Component placement
Authoring and publish controls sit in Registry and Evaluation Gate. Runtime handling sits in AgentCore plus LLM Gateway, Guardrails, MCP Gateway, and RAG services.
Capability Registry AgentCore LLM Gateway
Evaluation tracks
Track 1 is deterministic pre-publish scoring. Track 2 is runtime sampled quality scoring and online evaluators over production logs.
Promptfoo Bedrock Eval Langfuse judge
Feedback loop
Observed runtime regressions feed backlog updates, then a new version is evaluated and republished. Improvement is versioned, not patched in place.
Regression triage Versioned changes Publish gate
