LLM Gateway — mandatory entry point
  • Non-bypassable — all model calls route through LiteLLM; SCP denies direct provider API calls from L4
  • LiteLLM routes to configured provider — Bedrock, Azure OpenAI, or Gemini — based on tenant config and model permissions
  • Tenant quota checked — burst and sustained rate limits; circuit breaker state read
  • Request tagged: tenant_id, brand, agent_id, session_id — cost record opened; prompt cache checked

Guardrails Service — enforces Safety Envelope
— INPUT —
  • Enforces the Safety Envelope version locked to this Capability Definition at publish (I6) — permitted content scope, PII handling, denied topics
  • PII detection — names, NI, bank details redacted or blocked before the prompt reaches the model; never logged in plaintext
  • Prompt injection / jailbreak patterns scanned — Bedrock Guardrails native detection
  • Denied topics per tenant config — RG triggers, jurisdiction content; blocked inputs return 400 + trace ID
— OUTPUT —
  • DLP scan — no credentials, keys, or PII in response. Coverage varies by provider: Bedrock Guardrails native; LiteLLM callback guardrails for Azure OpenAI and Gemini routes
  • Response streamed with citations; modifications logged as-modified in Audit Record

Agent Runtime (AgentCore) — Reasoning scope
  • Loads Published Capability Definition from Capability Registry — immutable system prompt, tool bindings, Safety Envelope version, Execution Model, Risk Tier
  • ReAct loop — LLM reasons over the request, selects whether to retrieve context (RAG) or call a tool (MCP), observes the result, and repeats until ready to respond
  • ABAC session tags (tenant_id, brand, permission_scopes) from Okta → IAM Identity Center → STS session propagate to all downstream calls — agent cannot override
  • Credentials injected at runtime by Secrets Manager — never present in the agent definition or environment variables

Context & Tools — two distinct paths
— RAG Services · L3 —
  • tenant_id filter injected by platform at query time — cross-tenant retrieval structurally impossible
  • Hybrid kNN + BM25 against Bedrock KB, reranked; chunks carry source URI and data classification label
  • Retrieves from pre-ingested knowledge — documents synced by background ingestion pipelines, not fetched live
— Operational Connectors · L3 —
  • Live REST / GraphQL / MCP calls at execution time — Jira, Confluence, GitHub, ServiceNow and other internal systems. Result returned in ReAct loop, nothing persisted
  • RFC 8693 token exchange — IAM session traded for scoped OAuth token per target system; TTL 15 min, never reused, user identity does not reach the external system
  • All calls transit via AWS PrivateLink — no public internet egress from agents

LLM Inference — LiteLLM routed
  • Assembled context — system prompt, RAG chunks, tool results — submitted via LiteLLM to selected provider (Bedrock, Azure OpenAI, or Gemini) with streaming
  • Provider selected at runtime by LiteLLM — tenant permitted model list, availability, and cost policy determine routing. Provider is a runtime variable, not a deployment decision
  • Where Bedrock: per-tenant guardrail_id and cross-region inference profile applied. Where Azure OpenAI or Gemini: LiteLLM callback guardrails apply — coverage equivalence is a pending design point
  • Token cost attributed to tenant / agent / session — circuit breaker updated; hard stop enforced at budget limit
