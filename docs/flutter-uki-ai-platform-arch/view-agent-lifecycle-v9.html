Gate rigour proportional to Risk Tier
  • Pipeline requires evaluation results attached to the version record before the Registry publish call. Options: Promptfoo, LangSmith, Bedrock Evaluation, pytest against Registry API
  • Test invocations use a restricted IAM role — explicit deny on production data store writes. MCP calls hit sandbox endpoints with fixture responses
  • Results permanently attached to version record — tamper-evident, cannot be detached
  • Guardrails and ABAC are platform-enforced at runtime — the evaluation gate covers capability quality, not platform safety

I1
Lifecycle must be Review state to accept publish call — Draft cannot transition to Published directly
I2
Safety Envelope must be declared and validated. Risk Tier must be declared. Registry rejects the publish call without both — no exception path
I4
R2 or R3 Risk Tier: Workflow Contract must be declared referencing a valid Process scope specification. Registry rejects R2/R3 publication without it
I6
Safety Envelope becomes immutable on transition to Published. Changes require a new version — no in-place override path exists

Team chooses destination — one or both tracks
  • Capability Definition immutable at Published — system prompt, Safety Envelope, tool bindings, Risk Tier, Execution Model version-pinned. What was evaluated is what runs (I6)
  • Release is a Registry API call from the pipeline — tracks are independent
  • AgentCore resolves the Published Definition at invocation time — no runtime override possible (I1)

🔒
Governance is platform-enforced — not process-enforced
Platform invariants I1–I6 hold regardless of which authoring path was used, which release track was chosen, or who approved the release. The Registry gate is a hard block — no governance declaration can be bypassed by a process shortcut. The Safety Envelope is immutable from publish onwards. Every Execution produces a synchronous, immutable Audit Record regardless of Risk Tier — the semantics change at R2/R3 (fail-closed, write-ahead, extended retention), but the record is never optional.
