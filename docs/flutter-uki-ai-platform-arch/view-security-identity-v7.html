IAM Identity Center
  • Receives SAML assertion from Okta, maps attributes to permission set

  • Tags are attached to the STS session itself — they ride on every AWS API call natively

Identity Context — IAM Session Tags (I3)
  • This is the platform Identity Context — established once at session start, immutable for the full duration of the Execution (I3)
  • tenant_id, brand, role, use-case — these four tags drive all ABAC decisions downstream
  • Attached to every AWS API call made within this session — cannot be stripped or spoofed
  • ABAC policies at Capability Registry, RAG, and tool bindings all evaluate against these tags

Timing relationship
Flow 1 completes once at session start and establishes the Identity Context — immutable for the Execution duration (I3). Flow 2 fires inside the Agent Runtime each time the ReAct loop decides to call a tool — potentially multiple times per request, each producing a fresh scoped token.

MCP Gateway — RFC 8693 exchange
  • Performs RFC 8693 OAuth token exchange — IAM session traded for a short-lived tool-scoped OAuth token
  • Token is scoped to this specific tool and operation only — nothing broader
  • 15 minute TTL maximum — never reused across tool calls or sessions
  • This is where the user identity boundary ends — external systems see only the scoped token
